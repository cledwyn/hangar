Your basic Android Activity

Absolute minimum needed for an Activity

Android activities are your screens

public class MainActivity extends Activity {
   /** Called when the activity is first created. */
    @Override
    public void onCreate(Bundle savedInstanceState) {
        super.onCreate(savedInstanceState);
	setContentView(R.layout.activity_main);
    }
}
  • extends tells Android we are extending a class of Type Activity
  • You may also use implements like OnClickListener - implements tell Android to implement an action, like the mentioned listener inside your activity

<?xml version="1.0" encoding="utf-8"?>

  • Pro-TIP: You may use multiple layouts for a single screen based on device resolution and whether it is currently in portrait or landscape
  • Please see the samples for more advanced XML layouts

Android Activity Lifecycle

Activity with onResume, onPause and onDestroy

// called when the activity is drawn or redrawn...
	@Override
	public void onResume() {
		super.onResume();
		// check for updates to SharedPreferences, resume GPS etc...
	}
// called when the activity is no longer fully visible - stop GPS, etc.
	@Override
	public void onPause() {
		super.onPause();
	}
// called when the activity is being closed, set objects to null
	@Override
	public void onDestroy(){
		super.onDestroy();
		if(objecta != null) objecta = null;
		// release handle to GPS, msg handlers, etc.
	}
  • @Override's are used on any system wide methods when implemented
  • Example overrides are onCreate, onClick, onCreateOptionsMenu, onOptionsItemSelected, onPause, onStop, onResume, onDestroy etc...

The all important try-catch-Exception Block

Wrapping your code to handle Errors

	try {
	     enc = flipperCrypto.encrypt(ti, Constants.remoteSeed);
	} catch (Exception e) {
	     // TODO Auto-generated catch block
	     e.printStackTrace();
	}
  • Pro-TIP: The NullPointerException is the #1 error in Android development - be sure to wrap any code where an object may be null in a try-catch and make sure your objects/variables are instantiated FIRST
  • Pro-TIP: To catch an OutOfMemoryError use catch (OutOfMemoryError e) as it is not an exception but rather a System error - this should be used in ALL bitmap operations

One last Pro-TIP

Leveraging System.out.println(""); - It's useful in and out of try-catch blocks

	String enc = "";
	try {
	     enc = flipperCrypto.encrypt(ti, Constants.remoteSeed);
	     System.out.println("enc is: " + enc + " ti is: " + ti);
	} catch (Exception e) {
	     System.out.println("enc is: " + enc + " ti is: " + ti);
	     e.printStackTrace();
	}
  • By leveraging the built in System.out.println("") - you may debug your application in real-time on a real device while watching output in logcat
  • Place these anywhere you want output from your application for ease of development and debugging

OPTION 1 and 2 - Receiving Location updates

Lets setup our getLocationLatLng() method.

Download: goo.gl/ENs8l
    




// In your MainActivity.java, add the following below your other private variables at the top
private LatLng latLng; // right click LatLng to import
private double latC = 0;
private double lonC = 0;



// In your MainActivity.java, add the following inside your main class
private void getLocationLatLng() {
    try {
	if(locationManager != null) {
	latC = locationManager.getLastKnownLocation(LocationManager.GPS_PROVIDER).getLatitude();
	lonC = locationManager.getLastKnownLocation(LocationManager.GPS_PROVIDER).getLongitude();
	latLng = new LatLng(latC, lonC); // right click LatLng to import
	}
    } catch (Exception ex) {
	System.out.println("Oops, we had an error: "+ ex.Message());
    }
}
